Basierend auf meiner Analyse hier ist der Plan, um IEEE mit separaten Verzeichnissen für Online- und KI-Quellen zu verwenden:
Analyse des Problems
Herausforderungen:
1. IEEE unterstützt @online nicht - IEEE verwendet @electronic oder @misc
2. IEEE kennt keywords nicht - KI-Quellen müssen anders gefiltert werden
3. FOM-spezifische Felder wie usera werden ignoriert
4. Aktuelle Bibliographie-Filter funktionieren nicht mit IEEE
Lösungsplan
Schritt 1: BibTeX-Einträge an IEEE anpassen
Online-Quellen umwandeln:
% Von @online zu @electronic ändern
@electronic{website:example:tech,
    title = {Moderne Webtechnologien im Überblick},
    author = {Becker, Sascha},
    url = {https://www.example-tech.com/webtechnologien},
    year = {2021},
    month = {3},
    urldate = {2021-06-01}
}
KI-Quellen umwandeln:
% Von @misc zu @electronic und keywords entfernen
@electronic{KI_Claude_2025_Taxonomie,
    author = {{Anthropic Claude 4 Sonnet}},
    title = {Erstellung einer tabellarischen Taxonomie-Übersicht für Literaturrecherche},
    year = {2025},
    month = {8},
    url = {https://claude.ai},
    note = {Generiert am 02.08.2025, siehe Anhang KI.1}
}
Schritt 2: IEEE-Konfiguration erweitern
IEEE-Style in thesis_main.tex anpassen:
\ifthenelse{\equal{\citationstyle}{ieee}}{
    \usepackage[
        backend=biber,
        style=ieee,
        maxcitenames=3,
        maxbibnames=999,
        date=iso,
        seconds=true,
        urldate=iso,
        dashed=false,
        autocite=inline,
        useprefix=true,
        mincrossrefs=1,
        % IEEE-spezifische Optionen
        sorting=nyt,
        defernumbers=true
    ]{biblatex}

    % IEEE-spezifische Strings definieren
    \DefineBibliographyStrings{ngerman}{
        andothers = {{et\,al\adddot}},
        urlseen = {Zugriff am},
        urlfrom = {Verfügbar unter}
    }
}
Schritt 3: Bibliographie-Filter anpassen
Neue Filter-Strategie für IEEE:
% Hauptliteraturverzeichnis (nur Bücher und Artikel)
\printbibliography[type=book,heading=bibintoc,title={\langde{Literaturverzeichnis}\langen{Bibliography}}]
% Online-Quellen (electronic)
\newpage
\printbibliography[type=electronic,notkeyword=ai,heading=subbibliography,title={\headingNameInternetSources}]
% KI-Quellen (über note-Feld filtern)
\newpage
\defbibfilter{ai-sources}{
    note field [ai] or note field [KI] or note field [Claude] or note field [GPT]
}
\printbibliography[filter=ai-sources,heading=subbibliography,title={\langde{KI-generierte Quellen}\langen{AI-generated Sources}}]
Schritt 4: KI-Quellen konsistent kennzeichnen
Standardisierte note-Felder für KI:
note = {AI-generated content, siehe Anhang KI.1}
note = {KI-generierter Inhalt, siehe Anhang KI.2}
note = {Generated by Claude 4 Sonnet, see Appendix KI.3}
Implementierungsfragen
Frage 1: Möchtest du alle Online-Quellen automatisch zu @electronic konvertieren, oder nur bestimmte?
Frage 2: Soll die KI-Filterung über das note-Feld erfolgen (robuster für IEEE) oder möchtest du eine andere Strategie?
Frage 3: Behalten wir die deutsche Formatierung bei (Zugriff am, Verfügbar unter) oder soll es komplett englisch sein?
Frage 4: Sollen die KI-Quellen weiterhin im Anhang dokumentiert werden, oder reicht die Bibliographie?
Dieser Plan stellt sicher, dass IEEE-kompatible Einträge erstellt werden, während die Funktionalität für separate Verzeichnisse erhalten bleibt.
